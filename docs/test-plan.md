## Affected Pages/Routes
- Mobile: Run screen (live map, punch flow) — core loop; offline behavior is the point
- Mobile: Finish/splits screen — per-leg ranks, verification status + reasons
- Mobile: Course leaderboard — verified ranked, unverified shown unranked below
- Admin web: flag CRUD + photo, course authoring, missing-flag report inbox

## Key Interactions to Verify
- NFC punch (mock provider in E2E): leg clock stops at tag read; bottom sheet shows leg split vs best
- QR fallback and manual short-code entry demote the leg to partial, visibly
- Report-missing-flag button files a report; flag shows warning marker afterwards
- Sync after airplane-mode run: statuses computed server-side match app's provisional ones

## Edge Cases
- Interrupted sync retried → no duplicate runs/punches (idempotency)
- Two runners tap the same tag, sync in reverse order → both punches validate
- App kill / reboot mid-run → resume flow; affected legs partial, data never lost
- Track gap > 30 s, speed ceiling breach, proximity failure → correct demotions with reasons
- Client token attempts to write status columns → rejected by RLS

## Critical Paths
- Full verified run end-to-end: select course → punch all flags (NFC) → finish → splits → synced → ranked on leaderboard as verified
- Full offline run (airplane mode start to finish) → later sync → identical outcome
- Spoof trio ends not-verified: shared QR photo; GPX upload without punches; spoofed GPS without NFC
